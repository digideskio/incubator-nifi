Description:

This processor reads CSV files and inserts each CSV row as a record in a Kite dataset.

Properties:

Required properties are in bold.

  • Hadoop Configuration Resources
    • A file or comma separated list of files which contains the Hadoop file system configuration. Without this, Hadoop will search the classpath for a 'core-site.xml' and 'hdfs-site.xml' file or will revert to a default configuration.
    • Default value: none
  • Target dataset URI
    • URI of the dataset where records are stored. View URIs are also allowed.
    • This property supports the NiFi expression language.
    • Default value: none
  • CSV charset
    • Charset used to decode the incoming CSV files.
    • Default value: utf8
  • CSV delimiter
    • Delimiter character to separate CSV row fields.
    • Default value: ,
  • CSV quote character
    • Character used to quote CSV fields. Quoted fields can contain reserved characters, like new-line or the delimiter.
    • Default value: "
  • CSV escape character
    • Character used to escape characters in quoted CSV fields. The quote character must be escaped to be inside a quoted field
    • Default value: \
  • Use CSV header line
    • Boolean that signals whether incoming CSV files have a header in the first line (after any lines are skipped). If not present, the schema field order is used to decode a CSV row.
    • Default value: false
  • Lines to skip
    • The number of lines at the beginning of the file to ignore before reading a header (if present) and data rows.
    • Default value: 0

Relationships:

  • success
    • Files that were successfully appended to the dataset are sent to this relationship.
  • failure
    • Incoming files that could not be appended to the dataset are sent to this relationship.
